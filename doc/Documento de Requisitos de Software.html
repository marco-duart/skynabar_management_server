Documento de Requisitos de Software

1. Introdução

1.1 Objetivo do Documento

Este documento especifica os requisitos funcionais, não funcionais e regras de negócio para o sistema de controle de estoque do Pub Skyna Bar, baseado na norma ISO/IEC/IEEE 29148:2018 (versão simplificada). Ele servirá como base para o desenvolvimento, validação e manutenção do sistema.

1.2 Materiais de Referência

  • ISO/IEC/IEEE 29148:2018 - Requisitos de software e sistemas (Seções 5 a 8)
  • Notas da entrevista com o gestor do Pub Skyna Bar, 24/04/2025

2. Descrição Geral do Sistema

2.1 Propósito do Sistema

O sistema tem como propósito permitir o controle de estoque em tempo real, focando em simplicidade e acessibilidade mobile. Ele deve oferecer funcionalidades de registro de movimentações, gestão automatizada de reposição e controle de acesso baseado em perfil.

2.2 Escopo

Inclui:
  • Módulo de cadastro de produtos
  • Controle de entradas/saídas
  • Geração de lista de compras
  • Histórico de operações
Exclui:
  • Integração com sistemas fiscais
  • Controle financeiro
  • Gestão de pedidos de venda

3. Requisitos Específicos

3.1 Requisitos Funcionais

IDDescriçãoPrioridade
RF01Quando autenticado como Gestor, o sistema deve permitir o cadastro de produtos informando nome, unidade, quantidade atual e meta semanal.Alta
RF02O sistema deve registrar retiradas de produtos, com dados de produto, quantidade, usuário responsável e timestamp.Alta
RF03O sistema deve gerar automaticamente uma lista de compras com base na comparação entre estoque atual e meta semanal.Alta
RF04O sistema deve permitir ao Gestor estornar retiradas incorretas, mantendo o registro no histórico.Média

3.2 Requisitos Não-Funcionais

IDCategoriaDescrição
RNF01UsabilidadeA interface deve seguir diretrizes responsivas e acessíveis, permitindo uso intuitivo.
RNF02SegurançaO sistema deve exigir autenticação individual por usuário antes do uso de qualquer funcionalidade.

3.3 Regras de Negócio

IDDescrição
RN01Apenas usuários com perfil "Gestor" podem definir metas semanais.
RN02Apenas usuários com perfil "Gestor" podem estornar a saída de um produto.
RN03Apenas usuários com perfil "Gestor" podem cadastrar novos produtos.
RN04A lista de compras deve ser gerada com base na fórmula: (meta semanal - estoque atual).
RN05Apenas usuários com perfil "Gestor" podem visualizar o histórico de movimentações de estoque.

4. Restrições

  1. Plataforma primária: Web com design mobile-first
  2. Servidor baseado em sistema Linux
